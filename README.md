# 🖼️ AsyncImage Example

A SwiftUI app demonstrating how to load and display remote images asynchronously using `AsyncImage`.

---

## 🤔 What this is

This project shows how to use SwiftUI's built-in `AsyncImage` view to fetch and display images from a URL without any third-party dependencies. It covers placeholder states, error handling, and basic image styling — all with native SwiftUI APIs.

## ✅ Why you'd use it

- **No dependencies** — `AsyncImage` is built into SwiftUI; no Kingfisher or SDWebImage needed
- **Phase handling** — learn to handle loading, success, and failure states cleanly
- **Practical** — remote image loading is required in nearly every real-world app

## 🚀 Getting Started

### 1. Clone the Repo
```bash
git clone https://github.com/NDCSwift/AsyncImageExample.git
cd AsyncImageExample
```
Or select "Clone Git Repository…" when Xcode launches.

### 2. Open in Xcode
- Double-click `AsyncImageExample.xcodeproj`.

### 3. Set Your Development Team
In Xcode, navigate to:
**TARGET → Signing & Capabilities → Team**
- Select your personal or organizational team.

### 4. Update the Bundle Identifier
- Change `com.example.MyApp` to a unique identifier (e.g., `com.yourname.AsyncImageExample`).

---

## 🛠️ Notes

- If you see a code signing error, check that Team and Bundle ID are set.
- If building for a device, ensure your provisioning profile supports the required capabilities.

## 📦 Requirements

- Xcode 14+
- iOS 15+ (AsyncImage requires iOS 15)
- Swift 5.7+
